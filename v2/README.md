## 🎯 نظرة عامة

**GT-SARARIM** تطبيق تعليمي ترفيهي مصمم خصيصاً للطفل العربي المسلم، يجمع بين التعلم والمرح في بيئة آمنة تعمل دون الحاجة إلى اتصال بالإنترنت.

| الخاصية | التفاصيل |
|---|---|
| الفئة المستهدفة | الأطفال من 4 إلى 12 سنة |
| اللغة | العربية الفصحى المشكّلة |
| المنصات | Linux · Windows · Android |
| التخزين | محلي (بدون خوادم خارجية) |
| الرخصة | GPL-3.0 مفتوح المصدر |

## ✨ المزايا

### 📚 المحتوى التعليمي
- **8+ قصص** مصنّفة في فئات: قصص الأنبياء · الأخلاق والقيم · علماء المسلمين · العلوم
- **نصوص مشكّلة بالكامل** لمساعدة الأطفال على تعلم القراءة الصحيحة
- **أسئلة تفاعلية** بعد كل قصة مع خلط تلقائي للخيارات
- **15+ سؤال** في مجالات: الدين · الرياضيات · العلوم · اللغة العربية · الهندسة
- **8 ألغاز** بتلميحات مخفية، مصنّفة بمستويات صعوبة

### 🎮 الأنشطة
| النشاط | الوصف |
|---|---|
| 🧠 مسابقة المعلومات | أسئلة متعددة الخيارات مع خلط تلقائي |
| 🖼 اختبار الصور | تعرّف على الصورة الصحيحة من خيارات SVG |
| 🔵 وصل النقاط | 6 أشكال SVG تفاعلية (نجمة · بيت · قمر · سمكة · زهرة · جمل) |
| 🎨 التلوين | 6 صور SVG للتلوين الحر مع لوحة ألوان |

### 🏆 نظام التحفيز
- نقاط تراكمية وإنجازات قابلة للفتح
- 4 مستويات: تمهيدي → مبتدئ → متوسط → متقدم
- مكافآت فورية عند الإجابات الصحيحة المتتالية
- ألعاب نارية وتأثيرات بصرية تحفيزية

### 🎵 أصوات الخلفية (أوف لاين بالكامل)
مولّدة برمجياً بـ Web Audio API، لا ملفات صوتية خارجية:
- 🌊 خرير المياه وتغريد الطيور
- 🌧 صوت المطر
- 🌿 نسيم الطبيعة
- 🎵 موسيقى هادئة
- إمكانية رفع أصوات مخصصة (mp3, ogg)

### 🔒 أمان وخصوصية
- جميع البيانات محفوظة محلياً فقط (بدون أي خوادم)
- كلمة مرور للوالدين مع سؤال استرداد مخصص
- وقت لعب يومي قابل للضبط لكل طفل
- قفل أقسام محددة حسب رغبة الوالدين
- تراجع 10 ثوانٍ عند حذف أي محتوى

## 🚀 التثبيت والتشغيل

### المتطلبات
- **Node.js** 18 أو أحدث: https://nodejs.org
- **npm** 9 أو أحدث (يأتي مع Node.js)

### تشغيل وضع التطوير
```bash
# فكّ الضغط وادخل للمجلد
cd gt-sararim-final

# تثبيت الحزم (مرة واحدة)
rm -rf node_modules package-lock.json
npm install

# تشغيل المتصفح على http://localhost:5173
npm run dev
```

### تشغيل كتطبيق سطح مكتب مباشرة
```bash
npm run build
npm run electron
```

## 📦 بناء الحزم

### Linux (AppImage + deb)
```bash
bash BUILD.sh linux
# النتيجة في: release/
# ├── GT-SARARIM-2.0.0.AppImage
# └── gt-sararim_2.0.0_amd64.deb
```

### Windows
```bash
bash BUILD.sh win
# النتيجة: release/GT-SARARIM Setup 2.0.0.exe
```

### Android (APK)

**المتطلبات الإضافية:**
```bash
# Java JDK 17
sudo apt install openjdk-17-jdk

# Android Studio من: https://developer.android.com/studio
# ثم متغيرات البيئة (~/.bashrc أو ~/.zshrc):
export ANDROID_HOME=$HOME/Android/Sdk
export PATH=$PATH:$ANDROID_HOME/platform-tools
```

**بناء APK:**
```bash
# APK تجريبي (للاختبار الفوري)
bash BUILD-ANDROID.sh debug
# → release/GT-SARARIM-debug.apk

# APK نهائي (للنشر)
bash BUILD-ANDROID.sh release
# → release/GT-SARARIM-release-unsigned.apk
```

**تثبيت على الجهاز مباشرة:**
```bash
# مع تفعيل USB Debugging على الهاتف
adb install release/GT-SARARIM-debug.apk
```

**توقيع APK للنشر:**
```bash
# إنشاء مفتاح التوقيع (مرة واحدة)
keytool -genkey -v -keystore keystore.jks -alias gtsararim \
  -keyalg RSA -keysize 2048 -validity 10000

# البناء مع التوقيع
KEYSTORE_PASS=كلمة_المرور bash BUILD-ANDROID.sh release
```

## 🛡 لوحة تحكم الوالدين

**الدخول:** شاشة تسجيل الدخول → لوحة الوالدين → كلمة المرور  
**المدة:** تبقى مدخلاً 5 دقائق دون إعادة كلمة المرور

### إدارة الأبناء
- إضافة حسابات مع صورة وكلمة مرور اختيارية
- تحديد وقت لعب يومي خاص بكل طفل
- متابعة النقاط والمستوى

### إدارة المحتوى
| القسم | الإمكانيات |
|---|---|
| القصص | إضافة · تعديل · حذف · معاينة · صور/رموز · أسئلة خاصة |
| الأسئلة | 4 حقول للخيارات · الأول هو الإجابة الصحيحة · تُخلط عند الاختبار · صورة اختيارية |
| الألغاز | حل + تلميح + صورة اختيارية · مستويات صعوبة |
| الفئات | إضافة فئات جديدة برموز تعبيرية وألوان مخصصة |

### الأصوات
- تفعيل/تعطيل صوت الخلفية
- ضبط مستوى الصوت
- رفع أصوات مخصصة (mp3, ogg)

### الخطوط
- Ubuntu Arabic (افتراضي)
- Noto Naskh Arabic
- استيراد خط مخصص (TTF, OTF, WOFF)
- يبقى الخط المستورد محفوظاً حتى تحذفه يدوياً

### الأمان
- **تغيير كلمة المرور** مع تأكيد
- **سؤال الاسترداد** مخصص (تاريخ الميلاد · اسم المدرسة ...)
- **نسيان كلمة المرور:** يُسأل سؤال الاسترداد → كلمة مرور جديدة
- **إعادة الضبط:** مع حفظ احتياطي لبيانات الأبناء

### النسخ الاحتياطية
- **تصدير:** زر "تصديرٌ" → ملف JSON
- **استيراد:** زر "استيرادٌ" → اختر ملف JSON
- **تراجع الحذف:** 10 ثوانٍ لاسترجاع أي محتوى محذوف

## 🔧 تخصيص المحتوى

### إضافة قصة جديدة
1. لوحة الوالدين → المحتوى → قصص → "إضافة قصة جديدة"
2. أدخل العنوان والنص (مع الشكل للمساعدة على القراءة)
3. اختر رمزاً تعبيرياً أو ارفع صورة
4. أضف أسئلة: الحقل الأول = الإجابة الصحيحة (تُخلط عند الاختبار)
5. اضغط "معاينة ▶" للتجربة قبل الحفظ

### إضافة فئة قصص جديدة
1. لوحة الوالدين → المحتوى → قصص → قسم "فئات القصص"
2. اضغط "إضافة" → اختر رمزاً من الشبكة البصرية → حدد اللون → احفظ

### إضافة سؤال بصورة
1. لوحة الوالدين → المحتوى → أسئلة → "إضافة سؤال جديد"
2. ارفع الصورة (ستظهر فوق نص السؤال في الاختبار)
3. الخيار الأول = الإجابة الصحيحة دائماً

## 🐛 المشاكل الشائعة

**المشكلة:** `npm install` يفشل بـ ENOTEMPTY  
**الحل:** `rm -rf node_modules package-lock.json && npm install`

**المشكلة:** الخطوط لا تظهر في Electron  
**الحل:** تأكد من وجود الخطوط في `public/` وليس `fonts/`

**المشكلة:** الأصوات لا تعمل  
**الحل:** بعض المتصفحات تحتاج تفاعلاً أولياً — اضغط أي مكان ثم فعّل الصوت

**المشكلة:** APK لا يُثبَّت  
**الحل:** فعّل "مصادر غير معروفة" في إعدادات الأندرويد

**المشكلة:** لا تعمل لوحة المفاتيح العربية في APK  
**الحل:** `sudo apt install ibus ibus-table-arabic && ibus restart`

## 📦 المحتوى المضاف يدوياً والتحزيم

### المحتوى الافتراضي (مدمج في الحزمة)
كل ما هو موجود في `src/store/useStore.ts` (القصص، الأسئلة، الألغاز الافتراضية) يُدمج تلقائياً في AppImage وdeb وAPK.

### المحتوى المضاف يدوياً
المحتوى الذي تضيفه من لوحة التحكم يُحفظ في:
- **Linux AppImage/deb:** `~/.config/gt-sararim/Local Storage/`
- **Android APK:** في تخزين التطبيق الداخلي
- **يبقى محفوظاً** بعد إغلاق التطبيق وإعادة فتحه

### نقل المحتوى بين أجهزة
```bash
# من لوحة التحكم → زر "تصديرٌ" → يحفظ ملف JSON
# على الجهاز الجديد → زر "استيرادٌ" → يرفع نفس الملف
```

### لجعل محتواك المخصص "افتراضياً" في كل تثبيت
أضفه مباشرة في `src/store/useStore.ts` ثم أعد البناء.

## 🏗️ إضافة محتوى في وضع التطوير وتضمينه في الحزمة

### نعم — المحتوى في الكود يُدمج في AppImage/APK

**المحتوى الافتراضي** الموجود في `src/store/useStore.ts` (القصص، الأسئلة، الألغاز، الصور كـ SVG) يتحوّل إلى JavaScript خالص عند البناء وتجده مدموجاً في:
- `release/GT-SARARIM-x.x.x.AppImage`
- `release/gt-sararim_x.x.x.deb`
- `release/GT-SARARIM-debug.apk`

### كيف تضيف قصة/سؤال/لغز للحزمة؟

**الطريقة 1: مباشرة في الكود (تُدمج في الحزمة)**
```typescript
// في src/store/useStore.ts → INITIAL_STORIES
const INITIAL_STORIES: Story[] = [
  // ... القصص الموجودة ...
  {
    id: 'my_story_1',
    title: 'عُنْوَانُ قِصَّتِي',
    content: 'نَصُّ الْقِصَّةِ...',
    // ...
  },
];
```
ثم `npm run build` → `npm run pack:linux` وستكون في الحزمة.

**الطريقة 2: عبر لوحة التحكم + تصدير** (لا تُدمج، تنتقل عبر JSON)
```bash
# 1. أضف المحتوى من لوحة التحكم
# 2. لوحة التحكم → زر "تصديرٌ" → احفظ الملف
# 3. على جهاز آخر: "استيرادٌ" → اختر الملف
```

### سير العمل الموصى به للمطوّر
```bash
npm run dev          # أضف محتواك عبر الواجهة
                     # ثم انسخ البيانات من DevTools → Application → LocalStorage
                     # أو استخدم زر التصدير ثم أضف JSON إلى الكود

npm run build        # بناء
npm run pack:linux   # حزمة بالمحتوى الجديد
```
